Issue Found:

Vulnerability: Incorrect boundary check in is_index_valid(). The condition index > 0 incorrectly excludes index 0, and index < size incorrectly excludes the upper bound case depending on the intended semantics.

Violated Property: ESBMC reports a failure of the assertion r2 == 1 because is_index_valid(9, 10) incorrectly returns 0.

Fixed Code:

#include <assert.h>

int is_index_valid(int index, int size) {
    /* Correct boundary: allow 0 â‰¤ index < size */
    if (index >= 0 && index < size) {
        return 1;
    }
    return 0;
}

int main(void) {
    int r1 = is_index_valid(0, 10);
    int r2 = is_index_valid(9, 10);
    assert(r1 == 1);  // index 0 is valid
    assert(r2 == 1);  // index 9 is valid
    return 0;
}


Why This Fix Works:

The root cause is an incorrect lower-bound check (index > 0), which makes valid indices (e.g., 0) invalid, and therefore causes unexpected return values.

Changing the condition to index >= 0 restores the correct range of valid indices (0 to size - 1). ESBMC no longer finds violations because the function now returns values consistent with the assertions in main().
