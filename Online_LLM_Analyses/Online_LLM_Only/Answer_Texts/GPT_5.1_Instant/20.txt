1) Issue Found:

Vulnerability / failure: The function check_password always returns 0, even when the input matches the secret. Thus, the post-condition assert(r == 1) in main is violated.

Violated property: Functional correctness — the implementation does not satisfy the intended behavior “return 1 on correct password”.

2) Fixed Code:
#include <assert.h>
#include <string.h>

int check_password(const char *input) {
    const char *secret = "secret";

    /* Reject if input length differs */
    if (strlen(input) != strlen(secret)) {
        return 0;
    }

    /* Accept only if full match */
    if (strncmp(input, secret, strlen(secret)) == 0) {
        return 1;
    }

    return 0;
}

int main(void) {
    int r = check_password("secret");
    assert(r == 1);
    return 0;
}

3) Why This Fix Works:

The original code compared only a prefix and never returned 1, making successful authentication impossible.

The fix enforces exact-length match and full string comparison, returning 1 only when input fully equals secret, thereby satisfying the assertion in main.
